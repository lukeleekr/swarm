KNOWN_LIMITATIONS_OR_QUIRKS_DISCOVERED:
1. **Agent tool scope**: The `Agent(subagent_type=..., team_name=..., name=..., prompt=...)` tool is a top-level-orchestrator-only capability. A general-purpose subagent spawned via the same harness gets TeamCreate, TeamDelete, and SendMessage but NOT Agent. This means Native AgentTeam's spawn step cannot be transitively delegated to a subagent — only the top-level Claude Code session can spawn Claude teammates.
2. **Implication for swarm skill**: When the swarm rocky skill runs as a subagent (e.g., invoked via Skill tool from an outer orchestrator), Phase 3.2.2's Native AgentTeam path is not reachable. The skill must either (a) run at the top level, or (b) fall back to bash-spawned `claude --dangerously-skip-permissions` panes via swarm_spawn_agent, accepting the trust-prompt and CLI cold-start tax the skill was designed to avoid.
3. **TeamCreate/TeamDelete DO work from subagent**: The team config file was created at ~/.claude/teams/rocky-eval9/config.json with a lead-agent entry, and TeamDelete successfully cleaned up the directory. So partial Native AgentTeam machinery is reachable — only the Agent spawn step is blocked.
4. **SendMessage available but no recipient**: SendMessage was in the toolset and would have worked, but with no teammates to message it could not be used for shutdown_request.
5. **No error string produced**: Because Agent is simply absent from the tool schema (not a runtime-rejected call), there is no EXACT error message to quote. The "error" is structural: the tool never appeared in the available-tools list for this subagent.
6. **Panes not visible**: Since no Agent spawn happened, no tmux panes were created for teammates. The tile command was a no-op for teammates (the orchestrator+subagent panes were the only ones present).
